# Database System Redesign: Relational to NoSQL (MongoDB)

## Overview
This project involved analysing, redesigning, and migrating a flawed relational database system, followed by exploring a NoSQL alternative using MongoDB.

## Objectives
- Critique existing database design
- Create a normalized relational schema (up to 3NF)
- Implement and migrate data using SQL
- Re-design data structure using MongoDB
- Compare relational vs NoSQL approaches

## Technologies Used
- MySQL / MariaDB
- SQL
- MongoDB
- JSON
- ERD Modelling Tools (e.g. Visual Paradigm)

## Key Tasks

### 1. Database Critique
- Identified normalization issues
- Analysed poor table structure and relationships

### 2. Database Redesign
- Created ERD with proper relationships
- Eliminated redundancy using 3NF

### 3. Implementation
- Built database schema using SQL
- Defined primary and foreign keys

### 4. Data Migration
- Transferred data from legacy schema
- Used SQL transformations and insert queries

### 5. Query Development
- Wrote complex queries with joins and aggregations
- Replicated business logic from old system

### 6. NoSQL Design (MongoDB)
- Converted relational data into JSON documents
- Designed document structure for performance and flexibility

### 7. MongoDB Queries
- Implemented equivalent queries using MongoDB

## Key Learnings
- Importance of normalization in relational databases
- Differences between SQL and NoSQL systems
- Trade-offs between structured and flexible data models
- Real-world database design and migration challenges
